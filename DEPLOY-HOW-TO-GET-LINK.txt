HOW TO GET YOUR ONE PUBLIC LINK (works on phone, tablet, PC)
=============================================================

I cannot create the link for you from here — you choose a free host and
they give you the URL after you publish once.

FASTEST (about 1 minute, no Git required)
-----------------------------------------
1. Open https://app.netlify.com/drop in a browser (create a free account if asked).
2. Drag your **portfolio** folder (repo root with index.html) onto the page, or connect the repo (see netlify.toml).
3. Netlify shows a live link like https://random-name-123.netlify.app
4. In Netlify: Site settings → Change site name → pick something like
   dinuja-portfolio → your link becomes https://dinuja-portfolio.netlify.app
5. Share that single HTTPS link anywhere.

OPTION B — Vercel (recommended with Git)
----------------------------------------
Import the GitHub repo in Vercel for automatic deploys on every push.
See VERCEL-DEPLOY.txt in this repo for the exact project settings.

OPTION C — GitHub Pages (good if you use GitHub)
-------------------------------------------------
1. Push this project to your GitHub repo (main branch; **index.html** at the repo root).
2. Repository → Settings → Pages → Build and deployment.
3. Under "Source", choose "Deploy from a branch".
4. Branch: main — folder: **/** (root) — Save. (See SETUP-GITHUB-PAGES.txt in this repo.)
5. Wait 1–3 minutes for the first build.
6. Your site URL is usually:
   https://YOUR_USERNAME.github.io/YOUR_REPOSITORY/

Keep index.html and data.json in sync if you edit data.json and use
file:// locally; on the live site, data.json is loaded over HTTPS when
the inline block is removed or for updates — your current page already
embeds JSON in index.html so the site works without a build step.

After deployment, test the link on your phone (mobile data) to confirm.
