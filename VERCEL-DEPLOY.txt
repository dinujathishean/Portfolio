================================================================================
DEPLOY THIS PORTFOLIO ON VERCEL (one public link, works on phone + desktop)
================================================================================

This repo is static HTML/CSS/JS under **project/** (no npm build). Root **vercel.json** maps `/` to **/project/** when Root Directory is **`.`**.

STEPS
-----
1. Sign in at https://vercel.com (GitHub login is fine).

2. Click "Add New…" → Project → Import your GitHub repository:
   dinujathishean / Portfolio

3. Configure the project:
   - Framework Preset: **Other** (or "Other" / no framework)
   - Root Directory: **.** (leave default)
   - Build Command: **leave empty** (or put: `echo build skipped`)
   - Output Directory: **leave empty**

4. Click **Deploy**. Vercel prints your live URL, e.g.:
   https://portfolio-xxx.vercel.app
   You can rename the project under Project Settings → Domains / project name.

5. Every `git push` to the connected branch redeploys automatically.

GITHUB ACTIONS (failed "Configure Pages")
-----------------------------------------
If you still see an old workflow "Deploy portfolio to GitHub Pages" failing:
- Repo → **Actions** → select the workflow → **…** → **Disable workflow**
  OR delete `.github/workflows/<that-file>.yml` from the repo (this project ships
  without that workflow — use Vercel and/or GitHub Pages **Deploy from branch → main**).

For GitHub Pages without Actions, see SETUP-GITHUB-PAGES.txt in this repo.
