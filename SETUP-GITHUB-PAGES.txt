FAILED DEPLOYMENTS (red X, "Failed to deploy to github-pages ... Deploy portfolio...")
-------------------------------------------------------------------------------------
Those entries are from when GitHub Pages was set to build with **GitHub Actions**
and the workflow could not finish (or the workflow was later removed).

They will NOT fix themselves. You must stop using "GitHub Actions" as the Pages
source and publish the site straight from your **main** branch instead:

1. Open: https://github.com/dinujathishean/Portfolio/settings/pages
2. Under "Build and deployment" → **Source** → choose **Deploy from a branch**
   (NOT "GitHub Actions").
3. Branch: **main** → Folder: **/** (root) → **Save**.
4. Wait 1–3 minutes, then open: https://dinujathishean.github.io/Portfolio/

Old failed deployments may still appear in History — that is normal. What matters
is that the site loads after the steps above.

This repository no longer ships a GitHub Actions workflow for Pages; updates
happen when you push to **main**.

If GitHub still shows an old workflow "Deploy portfolio to GitHub Pages":
  Repo → **Actions** → click that workflow → **...** menu → **Disable workflow**.
